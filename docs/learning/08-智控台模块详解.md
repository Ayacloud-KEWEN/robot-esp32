# 08 · 智控台七大模块详解（功能 + 配置指南）

> 你在智控台"模型配置"里看到的：语音活动检测、语音识别、视觉大语言模型、意图识别、语音合成、记忆、知识库——这一篇逐个讲明白：它是什么、在流水线里的位置、有哪些可选方案、怎么配、怎么调优。
> 结合你服务器的实际现状（2026-07-18 核实）。

## 0. 全局视角：一句话如何流经这七个模块

```
设备送来音频
  → ① VAD 语音活动检测      "现在有人在说话吗？说完了吗？"
  → ② ASR 语音识别          "说的是什么字？"（语音→文本）
  → ③ Intent 意图识别        "想干什么？闲聊/点歌/控制设备/退出？"
  → ④ Memory 记忆           "关于这个用户我知道什么？"（取出相关记忆）
  → ⑤ 知识库                "有没有相关资料？"（检索文档片段）
  →   LLM 大语言模型         综合以上生成回复（你已配 DeepSeek ✅）
  →  (③′ 需要看图时调 VLLM 视觉大模型)
  → ⑥ TTS 语音合成          文本→语音，流式发回设备
  → ④′ Memory 对话结束后总结存储新记忆
```

**配置的两个层级**（重要）：
- **模型配置页**（管理员）：管理每类模块的"供应商清单"——启用哪些、填 API key、设默认
- **智能体（角色）配置页**：给每个角色**挑选**用哪个 ASR/LLM/TTS/记忆/意图——同一服务器上不同角色可用不同组合

---

## 1. 语音活动检测（VAD）

**干什么**：在音频流里判断"有没有人声、什么时候开始说、什么时候说完"。它是整条流水线的闸门——判停太快会截断你的话，判停太慢则回复迟钝。

**当前配置**：SileroVAD（本地开源模型，免费，也是唯一/标准选项，一般不用换）。

**关键参数**（模型配置 → 语言活动检测 → SileroVAD）：

| 参数 | 默认 | 说明 |
|---|---|---|
| `threshold` | 0.5 | 人声判定灵敏度。环境嘈杂误触发多 → 调高（如 0.6）；总漏听轻声说话 → 调低 |
| `min_silence_duration_ms` | 200 | 静音多少毫秒算"说完了"。**说话爱停顿被打断 → 调大（如 500-700）**；追求响应快 → 调小 |

> 这是普通人最值得调的参数之一：如果感觉"话没说完它就抢答"，把 `min_silence_duration_ms` 加大。

## 2. 语音识别（ASR）

**干什么**：把语音转成文字。识别准确率直接决定后面一切环节的质量。

**当前配置**：FunASR（阿里开源 SenseVoice 模型，**本地运行**，免费、无限量、隐私好——就是树莓派上占 2.5G 内存的那位）。

**可选方案**（服务器内置 17 种，分两类）：

| 类型 | 代表 | 优缺点 |
|---|---|---|
| 本地 | **FunASR**（你在用）、VOSK、Sherpa | 免费无限量、断网可用、隐私；吃内存，树莓派上速度中等 |
| 云端 API | 豆包ASR/豆包流式、阿里云、腾讯、百度 | 识别快且准（流式版延迟更低）；要 key、按量计费/有免费额度 |

**怎么配**：模型配置 → 语音识别 → 启用某供应商并填 key → 在智能体里切换。
**调优建议**：先用 FunASR 跑；若感觉识别慢（说完到出字 >1.5s），试试豆包流式 ASR 对比——这是降低整体延迟最有效的一换。

## 3. 视觉大语言模型（VLLM）

**干什么**：让机器人"看得见"。设备带摄像头拍照 → 上传到服务器的视觉接口（8003 端口 `/mcp/vision/explain`）→ VLLM 分析图片 → 用语音告诉你。对应"摄像头之眼"创意（问"我手里拿的是什么？"）。

**当前配置**：未配置（模板里默认是智谱 ChatGLMVLLM）。

**可选方案**：
- **glm-4v-flash（智谱）**：免费的视觉模型，入门首选（注册智谱开放平台拿 key）
- Qwen-VL（阿里）、豆包视觉等：按量计费

**怎么配**：模型配置 → 视觉大语言模型 → 填 key 设默认。**注意**：只有带摄像头的设备用得上（你的 ePaper-1.54 无摄像头；将来机芯盒或 S3-CAM 板可用；也可以走 go2rtc 家用摄像头的自定义插件路线）。

## 4. 意图识别（Intent）

**干什么**：判断你这句话是"闲聊"还是"要做事"（点歌、查天气、控制家电、退出对话……），决定要不要调用插件/工具。**这是所有插件功能的总开关**——设为不启用，天气/点歌/Home Assistant 全部失效。

**当前配置**：`function_call` ✅（最优选择）。

**三种模式对比**：

| 模式 | 原理 | 优缺点 |
|---|---|---|
| `nointent` | 不识别意图 | 纯聊天机器人，插件全关 |
| `intent_llm` | 前置一次独立 LLM 判断 | 兼容不支持函数调用的模型；但每句话多一次 LLM 请求，**变慢** |
| **`function_call`**（你在用） | 利用 LLM 原生函数调用能力，按需调用工具 | **快、准**；要求 LLM 支持 function call（DeepSeek 支持 ✅） |

**怎么配**：智能体配置里选择；一般保持 function_call 不动。写自定义插件（WiZ 灯、提醒等）后，它们自动进入工具列表被按需调用。

## 5. 语音合成（TTS）

**干什么**：把 LLM 的文字回复变成语音。**音色 = 机器人声音的人格**，和提示词同等重要。

**当前配置**：EdgeTTS（微软，免费，音色多，质量不错；走微软服务器偶尔抽风）。

**可选方案**（内置 23 种）：

| 类型 | 代表 | 说明 |
|---|---|---|
| 免费云端 | **EdgeTTS**（你在用） | 零成本首选；不可克隆音色 |
| 付费云端 | 豆包TTS、阿里、腾讯、火山双流式 | 更自然、延迟低、情感丰富；按量计费 |
| 本地/可克隆 | CosyVoice、GPT-SoVITS、FishSpeech、树莓派上还躺着 MOSS-TTS-Nano | **可克隆任意音色**（比如你自己的声音）；部署较重 |

**怎么配**：模型配置 → 语音合成 → 选供应商/音色；智能体里每个角色可选不同音色。
**试音色**：EdgeTTS 里换几个中文音色（晓晓、云希等）立刻感受差异；追求"专属声音"以后上 CosyVoice 克隆。

## 6. 记忆（Memory）

**干什么**：跨对话记住用户——名字、生日、喜好、上次聊过什么。对话结束后由 LLM 总结要点存储，下次对话自动带入。**陪伴感的核心模块**。

**当前配置**：`nomem`（未开启——建议开启！）。

**三种模式**：

| 模式 | 数据存哪 | 说明 |
|---|---|---|
| `nomem` | 不存 | 每次对话都是"初次见面" |
| **`mem_local_short`**（推荐） | **本地服务器** | 用你自己的 LLM（DeepSeek）做总结，数据不出家门，免费 |
| `mem0ai` | mem0 云服务 | 超长记忆能力强；每月 1000 次免费，数据在第三方 |

**怎么配**：智能体配置 → 记忆模块选 `mem_local_short` 即可（无需额外 key，用你已配的 DeepSeek 做总结）。
**验证**：告诉它"我生日是X月X日"，结束对话，明天再问"我生日是哪天？"

## 7. 知识库（RAG）

**干什么**：给机器人外挂"资料库"。把文档上传 → 系统切块并向量化 → 提问时检索最相关片段塞给 LLM 参考。与记忆的区别：**记忆是"关于用户的"，知识库是"关于世界/领域的"**（说明书、家庭档案、育儿资料、你的博客……）。

**当前配置**：未使用（智控台已内置知识库管理页面，后端表结构就绪）。

**怎么用**：
1. 智控台 → 知识库管理 → 新建知识库 → 上传文档（对接 RAGFlow 引擎时需先部署 RAGFlow 服务并在参数里配置其地址与 key）
2. 在智能体里关联该知识库
3. 对话中问相关问题，LLM 会引用检索到的内容回答

**注意**：RAGFlow 是独立服务，比较吃资源（建议 8G+ 内存，树莓派上可以但要精打细算，或部署在别处）。建议放在记忆、插件都玩熟之后再上。

## 8. 给你的行动建议（按性价比排序）

| 优先级 | 动作 | 成本 |
|---|---|---|
| ★★★ | **记忆改为 `mem_local_short`**——一次点选，陪伴感质变 | 0 |
| ★★★ | 跑通对话后按体感微调 VAD `min_silence_duration_ms` | 0 |
| ★★ | EdgeTTS 里试几个音色，给角色选个合适的声音 | 0 |
| ★★ | 注册智谱拿免费 key 配上 glm-4v-flash（为将来的"眼睛"备好） | 0 |
| ★ | 若识别延迟不满意，试豆包流式 ASR 对比 | 少量 |
| ★ | 玩熟后再考虑：CosyVoice 克隆音色、RAGFlow 知识库 | 较高 |

## 9. 附：MCP 的配置与接入（四条通道）

智控台/服务器对 MCP 支持完整（代码在 `core/providers/tools/`），共四条通道：

| 通道 | 方向 | 用途 | 怎么配 |
|---|---|---|---|
| ① 设备端 MCP（device_mcp） | 设备 → 服务器 | 固件里注册的硬件工具（音量、灯、GPIO、舵机） | 无需配置，设备连接时自动上报工具列表 |
| ② 服务端 MCP 服务（server_mcp） | 服务器加载外部 MCP | 外挂能力：Home Assistant、文件系统、搜索、任意第三方 MCP | 树莓派 `/opt/xiaozhi-server/data/` 下创建 `.mcp_server_settings.json`（模板见仓库 `xiaozhi-server/mcp_server_settings.json`；支持 stdio / sse / streamable-http 三种传输），改完重启 server 容器 |
| ③ MCP 接入点（mcp_endpoint） | 外部工具 → 挂到某个智能体 | 每个智能体有专属 WebSocket 接入地址，可把 PC 工具/脚本作为 MCP 服务挂给它 | 需先部署官方配套 mcp-endpoint-server，参数管理填 `server.mcp_endpoint`；之后在角色配置页查看该智能体的接入点地址与工具列表 |
| ④ 服务端插件（plugins_func） | 内置 | 天气/新闻/点歌/hass 等 | 意图识别开 function_call 即用（已开 ✅） |

**推荐第一步（待办）：通道② 接入 Home Assistant**——树莓派上 HA 已在运行，一次接入全屋设备打包变成小智工具（WiZ 灯若已入 HA 则连插件都省了）：

1. HA（`http://192.168.1.29:8123`）→ 用户头像 → 安全 → 创建长期访问令牌；确认 HA 的 MCP Server 集成已启用
2. 创建 `/opt/xiaozhi-server/data/.mcp_server_settings.json`：
   ```json
   {
     "mcpServers": {
       "Home Assistant": {
         "command": "mcp-proxy",
         "args": ["http://192.168.1.29:8123/mcp_server/sse"],
         "env": { "API_ACCESS_TOKEN": "你的HA长期令牌" }
       }
     }
   }
   ```
3. `docker restart xiaozhi-esp32-server` → 对话验证："把客厅灯调暗"
