# 05 · 陪伴机器人定制实战

> 本篇目标：综合前四篇所学，完成"自建服务器 + 定制固件 + 定制人格与能力"的端到端项目，做出属于你自己的陪伴机器人。

## 0. 前置检查

进入本篇前你应该已经：✅ 能编译烧录固件（01/02）　✅ 本地服务器跑通、web 测试页对话成功（03）　✅ 改过提示词、写过插件（04）。缺哪个先补哪个。

## 1. 总体施工顺序

```
Step 1  产品定义        —— 给谁用？做什么？长什么样？
Step 2  服务器定制      —— 人格 + 插件 + 模型选型（纯软件，最先做）
Step 3  固件指向自建服务器 —— 打通"我的设备连我的服务器"
Step 4  硬件定制        —— 板级适配、外设、MCP 工具
Step 5  联调与打磨      —— 延迟、打断、表情、开机体验
Step 6  上线运维        —— 公网、认证、监控、OTA
```

## Step 1 · 产品定义（写下来，别跳过）

回答这几个问题并记录到 `docs/product.md`：

- **用户**：自己？孩子？老人？→ 决定人格、音色、安全边界
- **核心场景**：闲聊陪伴 / 提醒吃药 / 讲故事 / 控制家电 / 桌面宠物？挑 2~3 个做深
- **形态**：现成开发板（推荐先用它把软件做完）→ 再做定制外壳/PCB
- **硬件清单**：芯片（推荐 ESP32-S3 N16R8）、麦克风、功放喇叭、屏幕、电池、电机/舵机（如需动作）

## Step 2 · 服务器定制

1. **模型选型定稿**：按 04 篇实验结果，在延迟/质量/成本三角里选定 ASR/LLM/TTS 组合，写进 `data/.config.yaml`。
2. **人格提示词**：定稿角色设定（智控台方式则在网页配置智能体）。
3. **自定义插件**：为核心场景写插件，放 `plugins_func/functions/`。典型陪伴场景插件：
   - 吃药/日程提醒（配合定时任务）
   - 讲故事（可对接知识库）
   - 家电控制（Home Assistant 插件已内置：`hass_*.py`）
4. **声纹与记忆**：多用户家庭开启声纹识别；确认 Memory 模块工作正常。

## Step 3 · 固件指向自建服务器

方式一（推荐，不改代码）：`idf.py menuconfig` → Xiaozhi Assistant → **OTA URL** 改为
`http://你的服务器:8003/xiaozhi/ota/`，重新编译烧录。

方式二：编译时清除 NVS 中的旧配置（`idf.py erase-flash` 后重烧）。

验证：monitor 日志里看到设备请求你的 OTA 接口、拿到你配置的 WebSocket 地址、连接成功、对话由你的服务器日志同步滚动。**这是整个项目最关键的里程碑。**

## Step 4 · 硬件定制

### 4.1 新建板级适配

```
1. 在 xiaozhi-esp32/main/boards/ 复制最接近你硬件的板目录，改名 my-robot/
2. 修改 config.h：I2S 麦克风/功放引脚、I2C、按键、LED、屏幕 SPI 引脚
3. 修改板类 .cc：换用你的 Codec/Display 驱动类
4. 在 boards 的 Kconfig / CMake 注册新板型，menuconfig 选中，编译烧录
5. 逐个外设验证：先声音（能录能放）→ 再屏幕 → 再按键/LED
```

### 4.2 为机器人加"身体"（MCP 工具）

在板级代码里初始化电机/舵机/灯，然后在 MCP 服务器注册工具，例如：

| 工具名 | 功能 | 说法示例 |
|---|---|---|
| `self.body.nod` | 舵机点头 | "同意的话点点头" |
| `self.body.turn` | 底盘转向 | "转个圈" |
| `self.light.set_mood` | 氛围灯颜色 | "来点温馨的灯光" |

工具描述写得越清楚，LLM 调用越准确。注意在提示词里告诉角色"你有身体，可以做这些动作"。

### 4.3 表情与音效定制

用官方 [xiaozhi-assets-generator](https://github.com/78/xiaozhi-assets-generator) 自定义表情包、字体、开机音效，生成 `assets.bin` 烧录。

## Step 5 · 联调与打磨清单

- [ ] 唤醒灵敏度合适（不误唤醒、不喊不应）
- [ ] 端到端延迟可接受（目标：说完到开始回答 < 1.5~2s）
- [ ] 打断自然（机器人说话时插话能立即停下来听）
- [ ] AEC 回声消除正常（机器人不会把自己的声音听进去）
- [ ] 表情/动作与对话情绪匹配
- [ ] 断网/服务器重启后能自动重连
- [ ] 长时间运行不发烫、不重启（观察数小时）

## Step 6 · 上线运维

- 公网部署 + wss + **开启认证**（见 03 篇），设备 MAC 加入白名单或走 token
- 服务器日志轮转、`docker compose` 开机自启（restart: always）
- 关注模型 API 用量与账单，设置额度告警
- 固件 OTA：新版本固件放到 OTA 服务，设备自动升级
- 用 git 打 tag 管理"固件版本 × 服务器版本"的配套关系

## 附：常见问题定位速查

| 症状 | 先查 |
|---|---|
| 设备连不上 | 设备 monitor 日志的 OTA 请求 → 服务器 8003 是否可达 → 防火墙 |
| 能识别不回话 | 服务器日志 LLM 环节：key、额度、网络 |
| 回话没声音 | TTS 环节日志 → 设备端功放引脚/音量 |
| 自问自答（回声） | AEC 配置、麦克风与喇叭距离 |
| 延迟大 | performance_tester 逐环节测，通常 LLM 首 token 是大头 |
