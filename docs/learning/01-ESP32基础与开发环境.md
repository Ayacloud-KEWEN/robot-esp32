# 01 · ESP32 基础与开发环境

> 本篇目标：理解 ESP32 是什么、掌握 ESP-IDF 开发环境，能独立编译烧录程序。

## 1. ESP32 是什么

ESP32 是乐鑫（Espressif）出品的**带 Wi-Fi/蓝牙的微控制器（MCU）**系列。它不是树莓派那样跑 Linux 的电脑，而是直接跑固件的芯片——程序烧进去、上电即运行。

### 1.1 常见型号对比（小智支持的）

| 型号 | CPU | 特点 | 在小智中的定位 |
|---|---|---|---|
| ESP32 | 双核 Xtensa 240MHz | 经典款 | 入门 |
| ESP32-C3 | 单核 RISC-V 160MHz | 便宜、小巧 | 低成本方案（虾哥 Mini C3） |
| ESP32-S3 | 双核 Xtensa 240MHz + AI加速指令 | 支持离线语音唤醒、可接屏幕摄像头 | **主流选择** |
| ESP32-C6 | RISC-V, Wi-Fi 6 | 新一代低成本 | 备选 |
| ESP32-P4 | 双核 RISC-V 400MHz | 高性能、无内置Wi-Fi | 高端（大屏、视觉） |

> 陪伴机器人推荐 **ESP32-S3（带 8MB PSRAM）**：离线唤醒词（ESP-SR）需要它的 AI 指令和大内存。

### 1.2 关键概念速览

- **GPIO**：通用输入输出引脚，控制 LED、读按键、驱动电机。
- **I2S**：数字音频总线，麦克风（如 INMP441）和功放（如 MAX98357A）用它传音频。
- **I2C / SPI**：外设总线，接屏幕、传感器、音频编解码芯片（Codec）。
- **PSRAM**：外挂大内存（8MB），音频缓冲、屏幕缓冲、AI 模型都靠它。
- **Flash 与分区表**：程序和资源存在 Flash 里，分区表规定每块区域干什么（见 `xiaozhi-esp32/partitions/v2/`）。
- **FreeRTOS**：ESP-IDF 自带的实时操作系统，提供"任务（Task）"= 轻量线程。小智固件里音频采集、网络收发、显示各是独立任务。

## 2. 开发环境：ESP-IDF

ESP-IDF 是乐鑫官方 SDK（C/C++，CMake 构建）。小智固件 v2 需要 **ESP-IDF 5.x**。

### 2.1 安装（Windows）

最简单：下载官方 **ESP-IDF Windows Installer**（espressif.com → ESP-IDF → Windows 安装器），它会装好工具链、Python 环境、驱动，并创建 "ESP-IDF PowerShell" 快捷方式。

安装后验证：

```powershell
# 在 ESP-IDF PowerShell 中
idf.py --version     # 应显示 v5.x
```

### 2.2 核心命令（必须熟练）

```powershell
idf.py set-target esp32s3    # 设置目标芯片（只需一次）
idf.py menuconfig            # 图形化配置（Kconfig）
idf.py build                 # 编译
idf.py -p COM5 flash         # 烧录（COM 口在设备管理器里看）
idf.py -p COM5 monitor       # 查看串口日志（Ctrl+] 退出）
idf.py -p COM5 flash monitor # 烧录+看日志，最常用
```

### 2.3 一个 ESP-IDF 项目长什么样

```
project/
├── CMakeLists.txt          # 项目级构建脚本
├── sdkconfig               # menuconfig 生成的配置（本仓库 xiaozhi-esp32/ 下就有）
├── sdkconfig.defaults.*    # 各芯片的默认配置
├── main/
│   ├── CMakeLists.txt      # 组件级构建脚本，列出源文件
│   └── main.cc             # 入口：void app_main()
└── managed_components/     # idf_component.yml 声明的第三方组件（自动下载）
```

对照本仓库：`xiaozhi-esp32/` 就是这个结构，入口在 `main/main.cc`。

## 3. 动手实验（过关标准）

### 实验 1：Blink（点灯）
新建 `idf.py create-project blink`，在 `app_main` 里用 `gpio_set_level` 让板载 LED 每秒闪烁。学会：GPIO 配置、`vTaskDelay`、编译烧录流程。

### 实验 2：串口日志与任务
用 `ESP_LOGI(TAG, "...")` 打日志；用 `xTaskCreate` 创建两个任务分别以不同周期打印。学会：FreeRTOS 任务概念、日志系统、`monitor` 用法。

### 实验 3：连接 Wi-Fi
用 ESP-IDF 的 `esp_wifi` 示例（`examples/wifi/getting_started/station`）连上家里 Wi-Fi 并打印 IP。学会：事件循环（event loop）、NVS（配置存储）。

### 实验 4：编译小智固件本体
```powershell
cd D:\xiaozhi20260403\xiaozhi-esp32
idf.py set-target esp32s3      # 按你的板子选
idf.py menuconfig              # 在 Xiaozhi Assistant 菜单里选板型
idf.py build
```
能 build 成功即过关（有板子就烧录试试）。

## 4. 常见坑

- **COM 口找不到**：装 CH340/CP210x 驱动；部分板要按住 BOOT 再插电进下载模式。
- **PSRAM 未启用导致崩溃**：menuconfig → Component config → ESP PSRAM 检查是否开启（小智 sdkconfig.defaults 已配好）。
- **编译报组件缺失**：删除 `managed_components/` 让其重新下载；检查网络（可配置乐鑫国内镜像）。
- **sdkconfig 混乱**：换板型/芯片后如异常，删掉 `sdkconfig` 和 `build/` 重新 `set-target`。
