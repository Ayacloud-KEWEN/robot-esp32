# 03 · 服务器与网络部署

> 本篇目标：掌握网络基础与 Docker，把 `xiaozhi-esp32-server` 在本地和公网跑起来，理解设备是如何找到并连上你的服务器的。

## 1. 需要掌握的网络基础

### 1.1 概念清单

| 概念 | 一句话解释 | 在小智中的体现 |
|---|---|---|
| IP / 端口 | 门牌号 / 房间号 | 服务器 8000(WS)、8003(HTTP/OTA)、8002(智控台) |
| HTTP | 一问一答的请求协议 | OTA 接口、视觉分析接口 |
| WebSocket | 在 HTTP 上升级出的**双向长连接** | 设备与服务器的主通道（音频流+JSON） |
| MQTT + UDP | 发布订阅消息 + 无连接数据报 | 另一种设备通道（低延迟音频走 UDP） |
| DNS / 域名 | 名字→IP 的翻译 | 公网部署时用域名代替 IP |
| SSL/TLS (wss/https) | 加密层 | 公网必备；`ws://`→`wss://` |
| NAT / 内网穿透 | 家庭网络没有公网 IP 的问题与解法 | 决定你部署在家里还是云服务器 |
| 反向代理 | 前置转发服务器（nginx/caddy） | 一个域名+443端口转发到多个内部服务 |

### 1.2 设备如何找到服务器（重要机制）

设备**不是直接配置 WebSocket 地址**，而是走两步：

```
1. 设备启动 → 请求 OTA 接口 (http://服务器:8003/xiaozhi/ota/)
2. OTA 响应里包含 WebSocket 地址（来自 config.yaml 的 server.websocket）
3. 设备用该地址建立 WebSocket 长连接
```

所以"把设备指向自建服务器"= 改固件里的 OTA URL（menuconfig 或代码），其余地址由服务器下发。

## 2. Docker 基础（部署的标准姿势）

Docker 把"程序+依赖+环境"打包成镜像，一条命令跑起来，不污染宿主机。

必会命令：

```bash
docker ps -a                 # 看容器
docker logs -f 容器名         # 追日志（排障第一步）
docker compose up -d         # 按 docker-compose.yml 启动全部服务
docker compose down          # 停止
docker compose pull && docker compose up -d   # 升级镜像
```

概念对应：**镜像**=安装包，**容器**=运行中的实例，**volume 挂载**=把宿主机目录映射进容器（配置和数据放宿主机，容器可随便删），**端口映射** `-p 8000:8000` = 宿主机端口→容器端口。

## 3. 实战：本地部署 xiaozhi-server

项目提供两种规模（见 `xiaozhi-esp32-server/main/xiaozhi-server/` 下的 compose 文件）：

- **最简部署** `docker-compose.yml`：只有 Python 核心服务，配置写 yaml 文件 → **先用这个**
- **全模块部署** `docker-compose_all.yml`：核心服务 + manager-api(Java) + manager-web(Vue) + MySQL + Redis，网页化管理

### 3.1 最简部署步骤

```bash
cd xiaozhi-esp32-server/main/xiaozhi-server
mkdir data
# 创建 data/.config.yaml，只写覆盖项，例如：
```

```yaml
# data/.config.yaml 示例
selected_module:
  LLM: ChatGLMLLM        # 选用的各环节供应商（名字须在 config.yaml 中存在）
  ASR: FunASR
  TTS: EdgeTTS
LLM:
  ChatGLMLLM:
    api_key: 你的key
```

```bash
docker compose up -d
docker logs -f xiaozhi-esp32-server   # 启动日志里会打印 OTA 地址和 WebSocket 地址
```

### 3.2 验证

1. 浏览器打开日志里的 OTA 地址（`http://本机IP:8003/xiaozhi/ota/`），能看到响应即 HTTP 通。
2. 用 `xiaozhi-web/` 测试页（`python -m http.server 8006` 托管）连接 WebSocket 地址，完成一次语音对话 → **本篇最重要的里程碑**。

### 3.3 排障思路

```
连不上？ → docker logs 看服务器是否启动成功
        → 防火墙是否放行 8000/8003（Windows Defender / 云安全组）
        → 设备和服务器是否同一局域网、IP 写对没有
没声音/不回话？ → 日志看卡在哪一环：ASR 没识别？LLM 报 key 错误？TTS 超时？
```

## 4. 公网部署（让机器人离家也能用）

### 4.1 方案选择

| 方案 | 成本 | 难度 | 说明 |
|---|---|---|---|
| 云服务器（阿里云/腾讯云 轻量 2C4G） | ~几十元/月 | ★★ | **推荐**：稳定、有公网IP |
| 家庭宽带 + 内网穿透（frp/Cloudflare Tunnel） | 低 | ★★★ | 适合折腾党 |
| 只在局域网用 | 0 | ★ | 陪伴机器人在家用其实够了 |

### 4.2 云部署要点

1. 安全组放行 8000/8003（或统一走 443 反向代理）
2. `data/.config.yaml` 里把 `server.websocket` 改成公网地址（`ws://域名:8000/xiaozhi/v1/` 或经反代的 `wss://域名/xiaozhi/v1/`）
3. 域名 + SSL：用 caddy（自动签证书，最省心）或 nginx + certbot 做反向代理，WebSocket 转发需保留 `Upgrade` 头
4. **务必开启认证**：`server.auth.enabled: true`，否则任何人都能连你的服务器烧你的模型额度
5. API key 只放在服务器的 `data/.config.yaml`，永远不进 git

## 5. 全模块部署（智控台）

跑通最简部署后再升级到 `docker-compose_all.yml`：多了 MySQL（存配置/设备/对话）、Redis（缓存）、manager-api、manager-web。此时配置模式切换为 `config_from_api.yaml`——**所有配置改为在智控台网页里管理**，yaml 不再生效。适合多设备、多角色、要看聊天记录的阶段。

## 6. 动手实验（过关标准）

1. 本地 Docker 最简部署成功，`xiaozhi-web` 完成语音对话。
2. 故意把 LLM 的 api_key 改错，从日志里找到报错位置——练排障。
3. （有条件）云服务器部署 + 域名 + wss，手机热点下用测试页连通。
4. 部署全模块版，在智控台里创建一个"智能体"并绑定测试。
