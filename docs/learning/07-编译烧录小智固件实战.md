# 07 · 编译烧录小智固件实战（按我的硬件定制版）

> 综合《Windows 搭建 ESP IDF 5.5》教程与本仓库实际情况，针对手上两块硬件的手把手操作指南。
> 硬件：① 微雪 ESP32-S3-ePaper-1.54 套件　② ESP32-S3 N16R8 机芯盒

## 0. 教程要点总结（原教程讲了什么）

1. **装环境**：从乐鑫官网 `https://dl.espressif.com/dl/esp-idf/` 下载 **IDF 5.5.3 离线安装包**（最新小智代码必须 ≥5.5.2，因为加了摄像头支持；本仓库 `idf_component.yml` 要求 `>=5.5.2` ✅）。别装 C 盘；**别用 VSCode 插件编译，用命令行**。
2. **验证**：装完双击桌面 "ESP-IDF 5.5 PowerShell"，用 `examples/get-started/hello_world` 试编译。编译时关掉杀毒软件（360/火绒/Defender）能大幅提速。
3. **编译小智四步**：`set-target` 选芯片 → `menuconfig` 里 Xiaozhi Assistant → Board Type 选板子 → `idf.py build` → `idf.py build flash monitor` 烧录看日志。
4. **常用技巧**：`-b 2000000` 提高烧录波特率；`idf.py build app-flash` 只烧 app 分区；`idf.py merge-bin` 合并成单个 bin（**给在线烧录用，见第 4 节**）；多板子用 `-p COM口` 区分；工程换路径要删 build；**工程别放中文路径**。
5. **唤醒词**：menuconfig → ESP Speech Recognition 里选现成的；或 Xiaozhi Assistant → Wake Word Implementation Type 选第三项 + Custom Wake Word 填拼音（如 `xiao ke xiao ke`）完全自定义。
6. **改服务器地址**：改 `main/Kconfig.projbuild` 里的 `OTA_URL`。**本仓库已改好指向 `http://robot.ayacloud.fr:8002/xiaozhi/ota/`**（你的智控台公网地址），无需再改。
7. **8MB 闪存的板子**（正是 ePaper-1.54 V2！）：menuconfig 改 Flash size 为 8MB + 分区表改 `partitions/v2/8m.csv`——好消息：本仓库板型配置 `config.json` 里 **v2 已自动带上这两项**，选对板型即可，无需手改。改过分区表务必 `idf.py fullclean`。
8. **AEC 语音打断**：需 ES7210 芯片的板子才支持（立创实战派、ESP-BOX 等），menuconfig 里手动开。
9. **其他**：串口卡住装 CH340/CP210x 驱动；I2C 冲突关 Audio Codec Device Configuration 第一行（新代码已修复）；表情包烧录 `esptool.py -p COM口 -b 2000000 write_flash 0x600000 assets.bin`。

## 1. 一次性准备（两块板子共用）

```powershell
# ① 安装 IDF 5.5.3（离线包，装到非C盘，如 D:\esp-idf）
# ② 打开 "ESP-IDF 5.5 PowerShell"，进入本仓库固件目录（注意：无中文路径 ✅）
cd D:\xiaozhi20260403\xiaozhi-esp32
# ③ 本仓库此前用旧版IDF编译过，先彻底清理一次
idf.py fullclean
```

> 板子插上 USB 后，在设备管理器 → 端口 里确认 COM 号（下文以 COM5 为例）。没有端口就装 CH340/CP210x 驱动；烧录失败可按住板上 BOOT 键再插电。

## 2. 实战 A：微雪 ESP32-S3-ePaper-1.54 套件

**先确认硬件版本**（V1=4MB Flash/2MB PSRAM，V2=8MB/8MB，套件外包装或用命令查）：

```powershell
esptool.py -p COM5 flash_id     # 看输出的 flash 容量
```

**编译烧录**：

```powershell
idf.py set-target esp32s3
idf.py menuconfig
#   → Xiaozhi Assistant → Board Type
#   → 选 "Waveshare ESP32-S3-ePaper-1.54"（构建变体 v2 或 v1 按你的硬件版本选）
#   → （可选）同菜单里设置自定义唤醒词
#   → 按 S 保存，Esc 退出
idf.py build
idf.py -p COM5 -b 2000000 build flash monitor    # 烧录+看日志，Ctrl+] 退出
```

> V2 板型已自动配好 8MB Flash + `partitions/v2/8m.csv` 分区表；OTA 已指向你的智控台。

**首次开机流程**：日志里看到设备起来 → 手机连它发出的 Wi-Fi 热点配网 → 设备请求 `robot.ayacloud.fr:8002` 的 OTA → 播报 6 位激活码 → 智控台"设备管理"输入激活码绑定 → 开始对话（流程与网页测试版相同，见[对话链路测试指南](../对话链路测试指南.md)）。

## 3. 实战 B：ESP32-S3 N16R8 机芯盒

N16R8 = 16MB Flash + 8MB PSRAM，默认 16m 分区表就是为它准备的，不用改 Flash 配置。

关键是 **Board Type 的选择取决于机芯盒的音频电路**（用什么麦克风/功放、接哪些引脚）：
- 如果是市售的"小智机芯盒"成品，查它说明书对应哪个板型（很多兼容"虾哥 Mini"或"面包板 WiFi"型）
- 如果是自接线，选 `Breadboard WiFi (bread-compact-wifi)` 并按其 `config.h` 的引脚定义接线
- 都对不上就到了我们的进阶课：在 `main/boards/` 下复制最像的板型建 `my-robot/` 自定义（见 [05 篇](05-陪伴机器人定制实战.md) Step 4）

```powershell
idf.py fullclean          # 换板型前清理
idf.py set-target esp32s3
idf.py menuconfig         # Board Type 按上面结论选，S 保存
idf.py -p COM5 -b 2000000 build flash monitor
```

## 4. 研究结论：能不能做"智控台网页在线烧录"？

**能做，而且分两种场景，其中一种现成就有：**

### 场景一：已刷过小智的设备升级固件 —— 智控台原生支持 ✅

智控台自带 **OTA 管理**（`manager-web` 的 `OtaManagement.vue`）：
1. 本地 `idf.py build` 后把固件 bin 上传到智控台 OTA 管理
2. 设备每次开机/定期请求 OTA 接口，发现新版本自动下载升级
3. **全程无需数据线**——这就是"在线烧录"的日常形态，我们应当把它跑起来作为标准发版流程

### 场景二：全新空白板子首次烧录 —— 需要开发，方案可行 ⭐

浏览器已具备直接烧录 ESP32 的能力：**Web Serial API**（Chrome/Edge 支持），成熟开源实现：
- **ESP Web Tools**（ESPHome 团队出品，esphome 官网"网页点一下就刷机"用的就是它）
- **esptool-js**（乐鑫官方的 JS 版烧录器）

**设计方案**（未来加到我们魔改的智控台里）：

```
[浏览器 Chrome] --USB线--> [空白ESP32板]
      │  Web Serial API (esptool-js / ESP Web Tools)
      ▼
智控台新页面 "固件烧录"
  ├─ 列出固件库（服务器上用 idf.py merge-bin 生成的 merged-binary.bin，
  │   合并版单文件从 0x0 烧录，正适合网页烧录）
  ├─ <esp-web-install-button> 一键：连接串口 → 擦除 → 烧录 → 看日志
  └─ 烧录完引导配网+绑定激活码
```

**约束条件**：
- Web Serial 仅 Chrome/Edge 支持（Firefox/Safari 不行）
- 页面必须是 **HTTPS 或 localhost** 才能调用 Web Serial —— 我们的智控台走 `robot.ayacloud.fr` 需要配 HTTPS 证书（可用 Nginx Proxy Manager 签，正好也是安全清单里想做的）
- 板子仍需 USB 线连到"操作者的电脑"（浏览器只能访问本机串口）——省去的是装 IDF 环境，不是省数据线
- S3 原生 USB 口即可，无需外置串口芯片 ✅

**实施路径建议**：先把场景一（OTA 升级）用起来 → 需要给新板子/朋友刷机时再开发场景二页面（前端加一个 Vue 页面 + 引入 esp-web-tools，后端加固件文件列表接口，工作量约 1-2 天）。已加入[将来可以做的事](../将来可以做的事.md)。
