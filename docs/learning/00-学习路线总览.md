# 00 · 学习路线总览

> 目标：通过"小智"项目，系统学会三大块知识——
> **① ESP32 嵌入式开发　② 服务器与网络部署　③ 大模型 AI 与陪伴机器人**
> 最终能独立完成：自建小智服务器 + 定制固件 + 定制陪伴机器人。

## 学习地图

```
第一阶段（打地基）
  01-ESP32基础与开发环境      ← 硬件是什么、ESP-IDF 怎么用、点亮第一个程序
        │
第二阶段（读懂固件）
  02-小智固件架构解析          ← 音频链路、事件循环、板级适配怎么写
        │
第三阶段（跑通服务器）
  03-服务器与网络部署          ← WebSocket/MQTT、Docker、公网部署、域名与SSL
        │
第四阶段（理解 AI 大脑）
  04-大模型与语音AI流水线      ← VAD/ASR/LLM/TTS、提示词、记忆、MCP工具调用
        │
第五阶段（综合实战）
  05-陪伴机器人定制实战        ← 人格设计、插件开发、硬件定制、端到端联调
```

## 建议节奏（可按自己情况调整）

| 周 | 内容 | 里程碑（做到才算过关） |
|---|---|---|
| 1-2 | 01 篇 | ESP-IDF 环境可用，独立编译并烧录一个 hello world / blink |
| 3-4 | 02 篇 | 能编译烧录小智固件；能说出一句语音从麦克风到服务器经过哪些模块 |
| 5-6 | 03 篇 | 本地 Docker 跑通 xiaozhi-server，用 xiaozhi-web 测试页完成一次语音对话 |
| 7-8 | 04 篇 | 换一个 LLM/TTS 供应商并对比效果；改提示词做出一个不同性格的角色 |
| 9-12 | 05 篇 | 固件指向自建服务器，真机对话成功；写出至少一个自定义插件 |

## 学习方法建议

1. **以本仓库代码为教材**：每篇文档都指向仓库里的真实文件，边看讲解边读源码。
2. **每个概念都动手验证**：改一行配置、加一行日志，观察行为变化，比看十遍文档有效。
3. **用 git 记录学习过程**：每完成一个实验就 commit，写清楚"我改了什么、观察到什么"。
4. **遇到问题的排查顺序**：看日志 → 缩小范围（是固件？网络？服务器？模型？）→ 最小化复现。
5. **随时问我**：我可以带你逐个文件精读源码、陪你调试、帮你设计实验。
