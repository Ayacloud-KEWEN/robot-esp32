## 2. 理解这台机器的"四层管理体系"

初学者最容易晕的是：同一台机器上，程序被四种不同的方式管理着。查问题时先想清楚目标属于哪一层：

```
① systemd（系统服务层）：Linux 自带的服务管家，开机自启的底层服务
     → frpc、nginx、docker、llama、go2rtc、tailscale、samba、pm2...
② Docker（容器层）：隔离打包的应用，大部分应用在这里
     → 小智全家桶、Home Assistant、Jellyfin、Portainer...
③ CasaOS（面板层）：网页版"应用商店"，它管理的其实也是 Docker 容器
     → 浏览器访问 http://192.168.1.29 就是它（80 端口）
④ pm2（Node 进程层）：Node.js 应用的进程管家
     → ayashop-front、strapi-shop
```

对应的"查状态"命令：

```bash
systemctl status frpc          # ① systemd 服务
docker ps                      # ② 容器列表
# ③ CasaOS 用浏览器看
pm2 list                       # ④ Node 进程
```

## 4. 常用操作速查（初学者背下来这些就够用 80%）

### 连接与基础

```bash
ssh fdcaptain@192.168.1.29     # 从 Windows 连上去（同内网）
ssh fdcaptain@100.74.246.122   # 出门在外走 Tailscale

htop                            # 看 CPU/内存（q 退出）
free -h                         # 内存概况
df -h                           # 磁盘概况
vcgencmd measure_temp           # CPU 温度
sudo reboot                     # 重启（约1分钟恢复，全部服务自启）
```

### Docker（最常用）

```bash
docker ps                       # 在跑的容器
docker logs -f --tail 50 xiaozhi-esp32-server   # 追某容器日志（排障第一步）
docker restart xiaozhi-esp32-server-web          # 重启某容器
cd /opt/xiaozhi-server && sudo docker compose -f docker-compose_all.yml up -d   # 按编排文件启动/更新小智全家桶
docker system df                # Docker 占了多少磁盘
docker image prune -af          # 清理未使用镜像（定期做）
```

### systemd 服务

```bash
systemctl status frpc           # 看状态（绿色 active 即正常）
sudo systemctl restart frpc     # 重启
journalctl -u frpc -f           # 追系统服务日志
systemctl list-units --type=service --state=running   # 列出所有在跑的服务
```

### pm2（Node 应用）

```bash
pm2 list                        # 列表（重点看 ↺ 重启次数，异常大说明在崩溃循环）
pm2 logs strapi-shop            # 看日志
pm2 restart ayashop-front       # 重启
```

### 小智专用

```bash
# 健康检查三连
curl http://localhost:8002/xiaozhi/ota/        # OTA 接口 → 应显示"运行正常"
docker logs --tail 30 xiaozhi-esp32-server     # 核心服务日志
docker ps | grep xiaozhi                       # 四个容器都应 Up

# 更新部署配置（改动走 GitHub 仓库）
cd ~/robot-esp32 && git pull
sudo cp ~/robot-esp32/deploy/pi/docker-compose_all.yml /opt/xiaozhi-server/
cd /opt/xiaozhi-server && sudo docker compose -f docker-compose_all.yml up -d
```

## 5. 以后部署新软件的建议（重要经验）

1. **优先用 Docker Compose，别裸装**。一个应用一个目录一个 `docker-compose.yml`，好装好卸好迁移。裸装的东西（如这台机器上的 nginx、frp、llama）时间久了自己都忘了怎么配的——你现在就深有体会 😄
2. **先查端口再部署**：`sudo ss -tlnp | grep 端口号`。这台机器端口已经很挤了（8000 被 Portainer 占导致小智被迫用 8005 就是教训）。建议新应用从 **18xxx** 段开始用。
3. **密钥永远走 `.env` 文件**（chmod 600），compose 里用 `${变量}` 引用——我们在 `/opt/xiaozhi-server/.env` 已经做了示范。
4. **大文件放 SSD**：媒体、模型、备份都放 `/mnt/ssd`（还有 715G 空闲），别塞 58G 的系统盘。
5. **部署完记录到 git 仓库**：像 `deploy/pi/` 那样把 compose 文件收编进仓库，写清楚端口和用途。三个月后的你会感谢现在的你。
6. **远程管理优先 Tailscale，公开服务才用 frp**：Tailscale 只有你自己的设备能访问（安全）；frp 是真公网暴露（要考虑认证）。管理面板类（8002、Portainer、CasaOS）建议只走 Tailscale。
7. **改配置前先备份**：`sudo cp xxx.yml xxx.yml.bak`，出问题一秒回滚。
8. **内存预算意识**：16G 中已用约 10G。大头：小智 ASR 2.5G、llama 2.45G、chromium 1.2G、HA+Jellyfin 各 0.6G。再上大内存应用前先算账。
