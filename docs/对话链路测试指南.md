## 1. 测试步骤

### 第 1 步：打开测试页

用 **Chrome** 打开：`http://192.168.1.29:3001`
（树莓派 nginx 托管的小智网页测试端；出门在外可走 Tailscale 访问同一页面）

### 第 2 步：确认服务器地址并连接

- 页面上的服务器/OTA 地址应指向：`http://192.168.1.29:8002/xiaozhi/ota/`
  （若页面填的是 `robot.ayacloud.fr` 公网域名也可以，走 frp 绕一圈而已）
- 点击连接，浏览器弹出麦克风权限请求 → **允许**

### 第 3 步：绑定设备（仅首次需要）

1. 新"设备"第一次连接不会直接对话，而是**语音播报一个 6 位激活码**（也会显示在页面/服务器消息里）
2. 另开标签页登录智控台：`http://192.168.1.29:8002`
3. 找到 **设备管理 → 添加/绑定设备**，输入播报的 6 位码
4. 绑定成功后，设备就挂在某个智能体（角色）名下了

### 第 4 步：开始对话 🎉

回到测试页，对着麦克风说话，比如：
- "你好，你叫什么名字？"（验证基本对话）
- "今天星期几？"（验证 get_time 插件）
- "北京天气怎么样？"（验证 get_weather 插件）
- 连续追问一个话题（验证多轮上下文）

**成功标准**：说完话 1~3 秒内听到语音回复，页面显示识别文本（stt）与回复文本。

## 2. 观察与记录（学习环节）

测试时开着服务器日志，能看到一句话流经整个流水线：

## 3. 常见问题排查

| 症状 | 排查 |
|---|---|
| 页面连不上 | 确认和树莓派同一内网；`docker ps | grep xiaozhi` 四个容器都 Up |
| 麦克风没反应 | Chrome 地址栏右侧麦克风图标 → 允许；系统声音设置里确认默认麦克风 |
| 播报激活码但智控台里找不到绑定入口 | 在"设备管理"或智能体详情页里找"绑定设备/新增设备" |
| 能识别文字但不回话 | `docker logs` 看 LLM 环节报错（key/网络/额度） |
| 回话没声音 | 浏览器标签页是否静音；日志看 TTS(EdgeTTS) 是否超时（EdgeTTS 走微软服务器，偶尔抽风，重试即可） |
| 延迟明显大 | 正常范围 1~3s；持续 >5s 则看日志定位卡在哪个环节 |

## 3.1 真机排障实录（2026-07-18，已解决）

**现象**：ePaper-1.54 上电后显示"检查新版本失败 Code=113（主机不可达），url=http://robot.ayacloud.fr:8002/xiaozhi/ota"。

**排查过程**：VPS 本机 curl 8002 = 200 ✅ → 外网走域名 = 000 ❌ → 直连 VPS IP = 200 ✅ → 查 DNS 发现域名解析到 Cloudflare 代理 IP（188.114.x.x）。
**根因**：Cloudflare 橙色云朵（Proxied）只转发标准端口（80/443/8080/8443…），**8002/8005 不被转发**。
**解决**：Cloudflare DNS 里把 `robot` 记录改为灰色云朵（DNS only）→ 域名直解 VPS → 设备断电重启（清 DNS 缓存）→ 激活码播报 → 绑定成功 → **真机对话跑通 ✅**。

**经验**：① 自定义端口的服务不能走 Cloudflare 代理，要么 DNS only，要么全部收编到 443 反代；② 设备报 113 时先分层测：本机 → 直连 IP → 域名，三步定位在哪一层断的；③ DNS 改动后设备要断电重启才会重新解析。
