## 2026-07-18 · 点歌插件优化（play_music.py）

**背景**：语音点歌跑通后发现三个体验问题：简体点歌匹配不到繁体文件名、模糊匹配经常命中错歌、找不到歌时静默随机播放（点《浮夸》放出一席演讲）。

**改动**（服务器端 `plugins_func/functions/play_music.py`，已从树莓派同步回仓库）：

1. **简繁归一化（最关键）**：引入 `zhconv`，搜索词和文件名统一转简体后再匹配。解决库里 `陳奕迅 -《浮誇》` 这类繁体文件名用简体"陈奕迅/浮夸"永远搜不到的问题。
   - 依赖安装位置：`plugins_func/vendor/`（**宿主机挂载目录**，随插件一起映射进容器，容器重建/升级镜像都不会丢）
   - 插件启动时把 vendor 目录加入 `sys.path` 再 import
2. **匹配算法重写**：弃用整串 difflib 相似度，改为**子串命中优先**——歌名只要是文件名的一部分即高分命中，不再被"林志炫 - "这类歌手前缀稀释分数；比较前统一忽略 `《》`、`-`、空格等符号。
3. **去掉静默随机兜底**：找不到时明确回复"抱歉，音乐库里没有找到《XX》"，不再随机抽一首。（"随便放一首"这类主动随机仍保留原逻辑）

**部署位置**：树莓派 `/opt/xiaozhi-server/plugins_func/`（compose 卷挂载，改完即生效，重启 server 容器加载）。
**曲库**：`/mnt/ssd/music`（129 首）→ 容器内 `music/` 目录。

**经验**：
- 插件目录是宿主机挂载的 → 改插件不用重建镜像，是最轻量的服务器定制层
- 给挂载目录配 `vendor/` 放第三方纯 Python 依赖，绕开"改容器镜像才能 pip install"的限制，升级镜像也不丢
- 中文曲库检索三件套：简繁归一化 + 符号归一化 + 子串优先
